# 数据溯源设计文档

## 核心原则

**原始数据永不丢失**：所有用户与 AI 的互动过程、修改记录、AI 建议、用户反馈都以原始形态保存，萃取只是"读取"而非"转换"，可随时重新萃取或更换 AI 模型。

---

## 数据库表结构总览

```
writer.db
├── materials              # 素材库
├── summaries              # 写作摘要（会话入口）
├── modification_sessions  # 修改会话（组织单元）
│
├── edit_records           # 【第一阶段】用户主动修改
├── revision_rounds        # 【第二阶段】打磨轮次
│
├── final_versions         # 定稿（清理后）
│
├── writing_preferences    # 通用偏好萃取结果
├── scenario_preferences   # 场景偏好萃取结果
└── writing_scenarios      # 场景定义
```

---

## 核心数据表详解

### 1. `summaries` - 写作会话入口

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | TEXT | 会话唯一标识 |
| `material_ids` | TEXT | 引用的素材 ID 列表（JSON） |
| `user_prompt` | TEXT | 用户原始请求 |
| `ai_original` | TEXT | AI 生成的初稿 |
| `final_text` | TEXT | 最终定稿文本 |
| `scenario_type` | TEXT | 场景类型（academic/daily_note 等） |
| `writing_purpose` | TEXT | 写作目的 |
| `created_at` | TIMESTAMP | 创建时间 |
| `modified_at` | TIMESTAMP | 最后修改时间 |

**用途**：每次写作的起点，链接素材和最终结果。

---

### 2. `modification_sessions` - 修改会话容器

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | TEXT | 会话 ID |
| `summary_id` | TEXT | 关联的写作会话 |
| `session_start` | TIMESTAMP | 开始时间 |
| `session_end` | TIMESTAMP | 结束时间 |
| `total_rounds` | INTEGER | 总修改轮数 |
| `is_finalized` | INTEGER | 是否已定稿 |
| `final_version_id` | TEXT | 关联的定稿记录 |
| `first_edit_recorded` | INTEGER | 首轮修改是否已记录 |

**用途**：组织一次完整的修改过程，关联 `edit_records` 和 `revision_rounds`。

---

### 3. `edit_records` - 用户主动修改（第一阶段）

**记录内容**：用户直接修改 AI 生成的文本

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | TEXT | 记录 ID |
| `summary_id` | TEXT | 关联的写作会话 |
| `session_id` | TEXT | 关联的修改会话 |
| `ai_original` | TEXT | AI 生成的原始文本 |
| `final_text` | TEXT | 用户修改后的文本 |
| `edit_ratio` | REAL | 编辑比例（0-1） |
| `diff_paragraph` | TEXT | 段落级 diff（JSON） |
| `diff_sentence` | TEXT | 句子级 diff（JSON） |
| `diff_word` | TEXT | 词语级 diff（JSON） |
| `is_final` | INTEGER | 是否被标记为最终稿 |
| `is_distilled` | INTEGER | 是否已被萃取 |
| `suggestions_raw` | TEXT | 原始建议列表 JSON（打磨轮次时） |
| `created_at` | TIMESTAMP | 创建时间 |

**数据安全性**：
- `ai_original` 和 `final_text` 保存完整原文，永不修改
- diff 数据独立存储，可随时重新计算
- `is_distilled` 仅标记是否被萃取过，不影响原始数据

---

### 4. `revision_rounds` - 打磨轮次（第二阶段）

**记录内容**：AI 提出建议 → 用户根据建议修改

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | TEXT | 记录 ID |
| `session_id` | TEXT | 关联的修改会话 |
| `round_num` | INTEGER | 轮次编号 |
| `ai_opinion` | TEXT | AI 提出的修改意见（完整文本） |
| `suggestion_type` | TEXT | 建议类型（结构/措辞/论据/精炼/场景） |
| `user_modified` | TEXT | 用户修改后的文本（**含 `%%...%%` 标记**） |
| `user_responded` | INTEGER | 用户是否响应（1=修改了，0=忽略） |
| `edit_ratio` | REAL | 编辑比例 |
| `suggestions_raw` | TEXT | **原始建议列表 JSON（含 anchor 定位信息）** |
| `user_feedback` | TEXT | **用户反馈 JSON（从 `%%...%%` 中提取）** |
| `created_at` | TIMESTAMP | 创建时间 |

**数据安全性**：
- `user_modified` 保存含 `%%...%%` 标记的完整文本
- `suggestions_raw` 保存 AI 生成的原始建议（含定位信息）
- `user_feedback` 保存用户填写的反馈，用于学习

**`suggestions_raw` 格式示例**：
```json
[
  {
    "problem": "这里读起来有点跳，缺少过渡",
    "advice": "加一句简短回答作为桥梁",
    "anchor": "段落 1"
  },
  {
    "problem": "论据不够具体",
    "advice": "补充实际案例或数据",
    "anchor": "段落 3"
  }
]
```

**`user_feedback` 格式示例**：
```json
[
  {
    "problem": "这里读起来有点跳，缺少过渡",
    "advice": "加一句简短回答作为桥梁",
    "user_feedback": "这是时事新闻，凡是关注这件事的人都知道，不用冗余添加",
    "responded": true
  }
]
```

---

### 5. `final_versions` - 定稿记录

**记录内容**：清理后的最终稿

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | TEXT | 定稿 ID |
| `summary_id` | TEXT | 关联的写作会话 |
| `final_text` | TEXT | **清理后**的最终稿（无 `%%...%%` 标记） |
| `total_rounds` | INTEGER | 总修改轮数 |
| `word_count` | INTEGER | 字数 |
| `created_at` | TIMESTAMP | 创建时间 |

**重要说明**：
- 定稿是清理后的版本，但**原始数据**（含 `%%...%%`）在 `revision_rounds.user_modified` 中完整保存
- 通过 `summary_id` 可回溯到完整的修改历史

| `scenario_type` | TEXT | 场景类型 |
| `version` | INTEGER | 版本号 |
| `preference_summary` | TEXT | 萃取摘要 |
| `system_prompt` | TEXT | 生成的系统 Prompt |
| `is_active` | INTEGER | 当前是否激活 |

## 数据流转图

```
用户输入 → summaries
              ↓
     AI 生成初稿 → ai_original
              ↓
     创建会话 → modification_sessions
              ↓
    ┌─────────┴──────────┐
    ↓                    ↓
第一阶段              第二阶段
edit_records      revision_rounds
（用户主动修改）     （AI 建议→用户修改）
    ↓                    ↓
    ├── ai_original      ├── ai_opinion（AI 意见）
    ├── final_text       ├── user_modified（含%%...%%）
    └── diff_*           ├── suggestions_raw（原始建议 JSON）
                         └── user_feedback（用户反馈 JSON）
              ↓
     定稿 → final_versions（清理后）
              ↓
     萃取 → writing_preferences / scenario_preferences
```

## 数据完整性保障

### 1. 原始数据永不覆盖

| 操作 | 数据处理方式 |
|------|-------------|
| 用户修改 | 新增 `edit_records` 记录，不修改已有记录 |
| AI 建议 | 保存到 `suggestions_raw`，不修改 `ai_original` |
| 用户反馈 | 保存到 `user_feedback`，不修改 `user_modified` |
| 定稿 | 新增 `final_versions` 记录，清理标记仅为展示 |
| 萃取 | 新增偏好记录，设置 `is_distilled` 标记但不删除数据 |

### 2. 可回溯性

通过以下链路可完整回溯任意一次写作过程：

```
final_versions.id
    → summary_id
        → modification_sessions
            → edit_records（所有修改记录，含 is_final 标记）
            → revision_rounds（所有打磨轮次，含完整%%...%%）
```

### 3. 可重新萃取

- `is_distilled` 字段仅标记是否被萃取过
- 删除偏好记录后，可重新运行萃取
- 更换 AI 模型后，可重新萃取生成不同的偏好

## AI 调用和学习机制

### 1. 实时调用（每次生成建议）

```python
from core.distiller import get_active_prompt

# 获取当前激活的偏好（含用户反馈规则）
prompt = get_active_prompt(prompt_type='writing', scenario_type='academic')

# 注入到 AI 请求
client.messages.create(
    model="claude-sonnet-4-5-20250929",
    system=prompt,
    messages=[...]
)
```

### 2. 用户反馈学习流程

```
用户填写反馈
    ↓
parse_user_feedback() 解析 JSON
    ↓
保存到 revision_rounds.user_feedback
    ↓
达到 5 条 → should_trigger_feedback_distill() 返回 True
    ↓
distill_user_feedback() 萃取
    ↓
分析拒绝/接受模式
    ↓
生成规则列表（如"时事新闻内容不要添加已知背景的过渡"）
    ↓
get_feedback_patterns() 返回规则
    ↓
合并到 get_active_prompt() → 注入下次 AI 生成
```

### 3. 萃取类型

| 萃取类型 | 数据源 | 触发条件 | 用途 |
|---------|--------|---------|------|
| 写作偏好 | `edit_records` | 3 条或 10 条 | 基础写作风格 |
| 场景偏好 | `edit_records` + `summaries` | 按场景 3 条或 10 条 | 场景化风格 |
| 修改过程偏好 | `revision_rounds` | 5 个会话 | 学习用户响应模式 |
| 定稿特征 | `final_versions` | 10 篇定稿 | 初稿质量优化 |
| 用户反馈偏好 | `revision_rounds.user_feedback` | 5 条反馈 | 避免用户拒绝的建议类型 |

## 安全边界

### 什么会被永久保存

- ✅ 用户原始输入（`summaries.user_prompt`）
- ✅ AI 生成的所有版本（`edit_records.ai_original`）
- ✅ 用户修改的所有版本（`edit_records.final_text`）
- ✅ AI 提出的完整建议（`revision_rounds.ai_opinion` + `suggestions_raw`）
- ✅ 用户填写的完整反馈（`revision_rounds.user_feedback`）
- ✅ 含 `%%...%%` 标记的原始草稿（`revision_rounds.user_modified`）
- ✅ 所有 diff 数据（段落/句子/词语）

### 什么会被清理

- ⚠️ 定稿文本（`final_versions.final_text`）- 清理 `%%...%%` 标记，但原始数据仍在 `revision_rounds` 中
- ⚠️ 草稿文件（`draft_*.md`）- 24 小时后自动删除，但数据库已保存

### 什么永不删除

- 所有 `edit_records` 记录
- 所有 `revision_rounds` 记录
- 所有 `user_feedback` 记录

---

## 更换 AI 模型

萃取机制与 AI 模型解耦：

1. 原始数据保存在 SQLite，不依赖任何 AI 服务
2. 萃取只是"读取"原始数据，生成偏好摘要
3. 更换 AI 模型后：
   - 删除 `writing_preferences` 中的偏好记录
   - 重新运行 `distill_preferences()`
   - 新 AI 会基于自己的理解生成新的偏好

## 数据查询示例

### 获取完整修改历史

```python
from core.revision_manager import get_session_history

history = get_session_history(summary_id)
# 返回所有 edit_records 和 revision_rounds
```

### 获取未萃取的反馈

```python
from core.feedback_distiller import get_feedback_records

records = get_feedback_records(limit=20)
# 返回包含 user_feedback 的打磨轮次
```

### 获取某会话的完整数据

```python
from core.revision_manager import get_session_edits, get_session_rounds

edits = get_session_edits(session_id)      # 所有编辑记录
rounds = get_session_rounds(session_id)    # 所有打磨轮次
```

## 总结

| 设计目标 | 实现方式 |
|---------|---------|
| **数据完整** | 所有原始文本、建议、反馈都以 JSON 或纯文本保存 |
| **可溯源** | 通过 session_id/summary_id 可回溯完整历史 |
| **可重新萃取** | is_distilled 仅标记，不删除；偏好表可删除重建 |
| **模型无关** | 数据保存在 SQLite，不依赖特定 AI 服务 |
| **用户反馈学习** | user_feedback 字段保存完整反馈，萃取后注入 Prompt |
